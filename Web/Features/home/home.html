Can I port my application to .NET Core?

It really depends on what you mean by .NET Core. Most commonly the term is used to refer to the .NET Standard Librarylaunch (.NET Standard). However it may also refer to the new tool stack.

The tool stack

The new .NET tool stack includes the dotnet command line toollaunch, xproj project structure and JSON configuration.

It is possible to take most existing applications, targeting .NET 4.5.1 or later and convert them to use this new structure. This is done by creating a new .NET Core project, editing the project.json file to change the frameworks section to target net451 (examplelaunch).

This website (sourcelaunch) uses the new tooling, but targets .NET 4.6.1 as not all our dependencies have been ported to .NET Standard. The new tooling supports targeting multiple platforms, so each project can be gradually be converted to also target .NET Standard when appropriate.

.NET Standard

.NET Standard Library is a set of APIs that are intended to be available on all .NET runtimes. Applications targeting .NET Standard should run on all platforms that implement that standard. There are two parts to assessing whether you can port your application.

The first is whether you use any APIs that are not part of the standard. For example not all of the types and methods in .NET 4.6 BCL are in the standard. Michael Whelan has written an excellent guidelaunch on analysing an existing project and porting it.

The second item to assess is whether there are .NET Standard version of your non-BCL dependencies available. That is where this website comes in.
